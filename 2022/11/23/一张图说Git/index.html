【一张图】Git QuickStar

发表于 2022-11-23 分类于 技术 阅读次数：

Git是先进的分布式版本控制系统，而Github是常用的Git代码托管中心

1 Git Workflow

sequenceDiagram
工作区->>暂存区:git add
暂存区-->>工作区:git rm
暂存区->>本地库A:git commit
本地库A-->>暂存区:git reset
本地库A->>远程库A:git push
远程库A-->>本地库A: git fetch
note left of 远程库A:git pull + merge
远程库A-->>远程库B:fork
远程库A-->>工作区:clone
远程库B-->>远程库A:pull request
note right of 远程库A:审核-->merge
  • 工作区：编写的区域
  • 暂存区：临时存储的区域
  • 本地库：保存版本信息

2 常用指令

  • git init：对所在目录进行本地库的初始化
  • git config [options] user.name <user-name>：配置用户名；
    • 项目级别的配置信息保存的位置为./.git/config
    • --global：设置为系统用户级别
  • git status：查看工作区与暂存区的状态
  • git add <file-name>：将工作区“新建/修改”的文件包含到暂存区以便commit
  • git rm --cached <file>：将暂存区的文件退回到工作区
  • git commit [options] <file>：将暂存区的内容提交到本地库
    • -m <context>：设置提交的消息
  • git remote [options]：设置远程仓库
    • -v：查看远程仓库
    • add <远程仓库别名> <远程仓库地址>：添加一个远程仓库，并为该远程仓库设定名字
  • git push <远程仓库别名> <分支名>：
    • git push origin master：将本地的 master 分支推送到 origin 主机的 master 分支
  • git clone <远程仓库地址>：克隆远程仓库（1.复制远程仓库，2.设置好远程的别名，3.初始化好本地仓库）
  • git pull <远程库别名> <远程分支>：fetch+merge；拉取远程库
    • git fetch <远程库别名> <远程的分支>
    • git merge <远程库别名/远程分支>
    • 如果多个push产生冲突，需要先pull下来，解决merge冲突
